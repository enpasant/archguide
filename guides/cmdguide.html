Introdução

O terminal é uma das ferramentas mais poderosas no Arch Linux. Com ele, você pode realizar uma grande variedade de tarefas, desde a instalação de pacotes até a administração do sistema. Aqui, você aprenderá alguns dos comandos mais úteis e como usá-los de forma segura.

Comandos Comuns

1. pacman

Uso: Gerenciador de pacotes do Arch Linux. Permite instalar, atualizar e remover pacotes do sistema.

Exemplo: sudo pacman -S nome_do_pacote

2. ls

Uso: Lista o conteúdo de um diretório.

Exemplo: ls /home

3. cd

Uso: Muda o diretório atual no terminal.

Exemplo: cd /home/usuario

4. sudo

Uso: Executa um comando com privilégios de superusuário.

Exemplo: sudo pacman -Syu

5. chmod

Uso: Altera as permissões de arquivos e diretórios.

Exemplo: chmod +x script.sh

6. top

Uso: Exibe os processos em execução no sistema em tempo real.

Exemplo: top

7. df

Uso: Exibe o uso do espaço em disco dos sistemas de arquivos montados.

Exemplo: df -h

8. grep

Uso: Pesquisa por padrões em arquivos de texto.

Exemplo: grep "erro" log.txt

Comandos Perigosos

1. rm -rf

Uso: Remove arquivos e diretórios recursivamente, sem pedir confirmação. Pode apagar todo o sistema se usado de forma errada.

Exemplo: sudo rm -rf /

2. mkfs

Uso: Cria um sistema de arquivos em uma partição. Apaga todos os dados da partição.

Exemplo: sudo mkfs.ext4 /dev/sda1

3. dd if=/dev/zero of=/dev/sdX

Uso: Sobrescreve um dispositivo de armazenamento (como um disco rígido) com zeros, apagando todos os dados de forma irreversível.

Exemplo: sudo dd if=/dev/zero of=/dev/sda

4. shutdown -h now

Uso: Desliga imediatamente o sistema, sem qualquer aviso. Pode causar perda de dados se estiver com arquivos não salvos.

Exemplo: sudo shutdown -h now

Aviso: Comandos perigosos, como rm -rf e dd, podem causar danos irreparáveis ao sistema e perda de dados. Sempre verifique duas vezes antes de executar comandos com sudo e use-os com extrema cautela. Certifique-se de fazer backup dos dados importantes regularmente.
